# Portfolio page publishing guide

This folder contains a simple static site that is ready for GitHub Pages.

## 1. Create a public GitHub repository

You have two good options:

- `your-username.github.io`
- `university-projects-portfolio`

If you want the cleanest URL, use `your-username.github.io`.

## 2. Upload these files

Upload the contents of this folder:

- `index.html`
- `styles.css`
- `README.md`

They should be in the root of the GitHub repository.

## 3. Enable GitHub Pages

In GitHub:

1. Open the repository.
2. Go to `Settings` -> `Pages`.
3. Under `Build and deployment`, choose `Deploy from a branch`.
4. Select the `main` branch and the `/root` folder.
5. Save.

After a minute or two, GitHub will give you a public URL.

## 4. Improve the page before publishing

Recommended final changes:

- Add your real GitHub profile link.
- Add public repository links for each project.
- Add 1 screenshot for each project if you have one.
- Add 1 line describing your individual contribution in each team project.

## 5. Best resume placement

For your resume, the best places are:

- In the header, next to your email and phone: GitHub profile + portfolio URL.
- A new section called `Selected Projects` between `Education` and `Experience`, or between `Experience` and `Technical Skills`.

Suggested short text for the header:

`GitHub: github.com/your-username | Portfolio: your-username.github.io`

Suggested project section entries:

- `Static Analyzer for Python Web Vulnerabilities` - Built a Python AST-based static analysis tool to detect insecure flows, sanitization paths, and multiple web vulnerability patterns.
- `ChainOfProduct Secure Document Platform` - Developed a secure multi-service system with FastAPI, Java cryptography, PostgreSQL, TLS, and certificate-based document sharing.
